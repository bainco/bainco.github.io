Alright, I implemented the same Peer Review assignment from the previous quarter and this time it went much smoother (mostly because I knew what would go wrong). From the results, I saw the same encouraging qualitative results in terms of student and course staff talk. However, I've also decided a few things:

1. This data isn't fit for a formal design publication (it could be used for work-in-progress paper but really shows some flaws in the implementation rather than the idea).
2. I want to continue doing this peer review assignment (and in fact want to do it in my other large format class, CS 111) but I can't rely on the Canvas built-in Peer Feedback tool.
3. I have some preliminary thoughts on the design of a tool integrated with Canvas that I could make which would allows students to:
  * see the output of a peer's code
  * comment on specific lines
  * more critically interact with the rubric
4. There is absolutely no way I'll be able to make this during the quarter. 
5. Instead of the "publication" deliverable for my final project, I'm going to pivot to releasing some of the Canvas API tools I developed to fix these issues so that others can use them if they want to attempt something similar.

On the other side of things, I did formally add our peer review learning objective to the syllabus this quarter and continued adding more and more "micro assessments" of these "reading" skills to the quizzes and mini-quizzes
